2) BACKEND CHANGES (Core APIs)
========================================================
Add/Update these endpoints:

POST /api/videos/import
- Body:
  {
    "sourceUrl": "string",
    "mode": "direct" | "provider",
    "title": "optional",
    "description": "optional",
    "author": "optional"
  }

Behavior:
1) Create video row in DB with status='processing' and source_type set:
   - direct_url
   - vimeo_ingest
   - youtube_blocked
2) If mode="direct":
   - Validate URL endsWith .m3u8 or .mp4
   - Save it as external_source_url (new column) BUT IMPORTANT:
     - For best security, if direct points to your own S3/Supabase, copy it into our storage OR require signed access.
   - If .mp4: enqueue transcode to HLS into our storage; status='processing'
   - If .m3u8: optionally “mirror” playlists/segments into our storage (recommended) else store as “external_hls_url” (less secure)
3) If provider="vimeo":
   - Attempt to resolve downloadable/progressive file URL via Vimeo API:
     - Requires VIMEO_ACCESS_TOKEN env var
     - Parse Vimeo video ID from URL
     - Call Vimeo API GET /videos/{id}
     - Look for files array: progressive mp4 or HLS link (if permitted)
   - If a downloadable MP4 is available:
     - Download/stream it server-side into OUR storage (S3/Supabase) under raw/{videoId}/source.mp4
     - Then run ffmpeg to generate HLS to hls/{videoId}/...
     - Update DB: raw_object_key, hls_master_key, status='ready'
   - If Vimeo does NOT provide direct downloadable file:
     - Set status='error'
     - Set last_error: “Vimeo file not available for ingest. Your Vimeo plan/privacy settings do not expose MP4/HLS. Upload instead.”

4) If provider="youtube":
   - Immediately set status='error'
   - last_error: “YouTube links are not supported for custom player playback. Upload the file instead.”

Return:
- { videoId, publicId, status, message }
